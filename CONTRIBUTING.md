# Contributing

Thanks for helping improve Lucky Draw Kit.

## Development

```bash
npm test
```

## Pull requests

- Keep changes focused.
- Add or update tests when changing draw behavior.
- Update documentation when adding public APIs.
- Avoid adding runtime dependencies unless the benefit is clear.

## Maintainer workflow

- Issues are triaged by bug, feature, docs, and question labels.
- Releases should include test results and a short changelog entry.
- Security-sensitive reports should follow `SECURITY.md` instead of public issues.
